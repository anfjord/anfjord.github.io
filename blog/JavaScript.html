Ruby Hashes vs. JavaScript Objects

Differences

  • In Javascript hashes are referred to as objects. The key in ruby is a variable in JS.
  • When you call a hash in ruby, you use the key. In JS you call the object to get the value of the key.
  • In JS, object can take a function as a value. Since methods in Ruby aren't objects, you can't assign a method as a hash value.
  • In JS there are no enumerable iterators, which you have in Ruby.
  • Semicolons. They are not used in statements in Ruby, while the are necessary to separate statements in JS

    • Similarities

  • Their syntax is very similar. Minus the semicolon.
  • They follow the same logic with pair values. In Ruby the label is called key, in JS property.
  • The values can be of any type; hashes, arrays, numbers, strings etc.
